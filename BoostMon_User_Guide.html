üéÆ The Problem with Roblox Private Server Boosts

Roblox private servers are expensive to maintain. To keep stat multipliers active 24/7, groups split the cost across multiple players contributing "boosts" on a rotating basis. This creates several challenges:

  • Tracking Chaos: Who boosted when? Who owes what? Spreadsheets get messy
  • No Accountability: Players forget when they're supposed to boost next
  • Manual Enforcement: Leaders manually remove roles when timers expire
  • Duplicates & Conflicts: Multiple admins managing roles leads to mistakes
  • Player Frustration: No clear visibility into how much time they have left
  • Server Downtime: Forgotten boosts mean stat multipliers die unexpectedly

‚ú® The BoostMon Solution

BoostMon automates everything. Set a timer when a player boosts, get automatic warnings as expiration nears, and remove inactive players without drama. Everyone knows exactly how much time they have until removal.

Key Benefits:

  • ‚úÖ Zero Manual Tracking: Database handles everything
  • ‚úÖ Transparent Timers: Players see exactly when they expire
  • ‚úÖ Automated Warnings: Reminders at 60, 10, and 1 minute marks
  • ‚úÖ Auto-Removal: Inactive players automatically lose roles
  • ‚úÖ Fair & Accountable: No room for mistakes or favoritism
  • ‚úÖ Always Available: Works 24/7, even if bot goes offline

5Ô∏è‚É£ Getting Started

Step 1: Add BoostMon to Your Server

  1. Visit top.gg and search for "BoostMon"
  2. Click the "Invite" button
  3. Select your Discord server from the dropdown
  4. Verify permissions (Manage Roles, View Channels, Send Messages)
  5. Click "Authorize"
  6. BoostMon is now ready to use!
‚úÖ Required Permissions:
  • Manage Roles
  • View Channels
  • Send Messages
  • Use Slash Commands

Step 2: Create Your First Timer

In any channel, type:

/settime @john 60 @VIP #announcements

This command:

  • Creates a 60-minute timer for @john
  • Assigns the @VIP role immediately
  • Sends expiration warnings to #announcements

Step 3: Configure Warning Notifications

Warnings are automatically sent at:

  • 60 minutes remaining
  • 10 minutes remaining
  • 1 minute remaining
  • 0 minutes (final removal notice)
üí° Tip: Omit the channel parameter to send warnings as DMs instead:
/settime @john 60 @VIP

Step 4: Check Timer Status

Anytime, use:

/showtime @john

See remaining time, expiration timestamp, and whether timer is paused.

6Ô∏è‚É£ Complete Command Reference

1. /settime - Create a Timer

Syntax: /settime @user <minutes> @role [#channel]

Parameters:

  • @user (required): Target Discord user
  • minutes (required): Duration in minutes (1-10,080)
  • @role (required): Role to assign
  • #channel (optional): Warning notification channel

Example:

/settime @player 1440 @Active-Booster #boost-tracking
Creates 24-hour timer, assigns role, sends warnings to #boost-tracking

2. /addtime - Extend Timer

Syntax: /addtime @user <minutes> [@role]

Example:

/addtime @player 480 @Active-Booster
Adds 8 hours to existing timer

3. /removetime - Reduce Timer

Syntax: /removetime @user <minutes> [@role]

Example:

/removetime @player 120 @Active-Booster
Removes 2 hours (partial refund scenario)

4. /cleartime - Delete Timer

Syntax: /cleartime @user [@role]

Example:

/cleartime @player @Active-Booster
Removes timer completely and deletes role

5. /pausetime - Freeze Timer

Syntax: /pausetime @user [@role]

Example:

/pausetime @player @Active-Booster
Pauses countdown. Role stays active. Time doesn't count down.

6. /resumetime - Unfreeze Timer

Syntax: /resumetime @user @role
‚ö†Ô∏è Important: Role parameter is REQUIRED to prevent ambiguity if user has multiple timers.

Example:

/resumetime @player @Active-Booster
Resumes from pause with remaining time

7. /showtime - Check Remaining Time

Syntax: /showtime @user [@role]

Example:

/showtime @player @Active-Booster
Shows: 45 minutes 30 seconds remaining

8. /rolestatus - View All Role Members

Syntax: /rolestatus @role

Example:

/rolestatus @Active-Booster
Lists all members with @Active-Booster and their remaining times

7Ô∏è‚É£ Use Cases & Real-World Examples

Use Case 1: Daily Boost Tracking (Most Common)

Track active boosters who contribute daily to your private server:

/settime @player-1 1440 @Active-Booster #boost-tracking /settime @player-2 1440 @Active-Booster #boost-tracking /settime @player-3 1440 @Active-Booster #boost-tracking

What Happens:

  • Each player gets a 24-hour timer (1440 minutes)
  • Warnings sent at 60, 10, and 1 minute marks
  • After 24 hours, role auto-removes if they haven't boosted again
  • Crystal clear: everyone knows their deadline

Use Case 2: Multi-Day Boost Packages

Track players who committed to multiple days:

/settime @committed-player 2880 @3-Day-Booster #boost-tracking

Result: 48-hour (2-day) timer with appropriate recognition role

Use Case 3: Refunds & Corrections

Player boosted but wants a partial refund:

/removetime @player 480 @Active-Booster

Result: Remove 8 hours (480 minutes) for partial refund

Or for complete refund:

/cleartime @player @Active-Booster

Use Case 4: Emergency Break

Player needs to take a break (vacation, emergency):

/pausetime @player @Active-Booster

Timer freezes. Role stays active. Time doesn't count down.

When they're back:

/resumetime @player @Active-Booster

Resumes from exactly where it paused.

Use Case 5: Status Check Before Expiry

Quick check to see who's expiring soon:

/showtime @player @Active-Booster /rolestatus @Active-Booster

See exact minutes remaining for any player.

8Ô∏è‚É£ Frequently Asked Questions

Q: Will my timers persist if the bot goes offline?

A: Yes! All timers are stored in PostgreSQL. They continue running even if BoostMon is offline. When the bot comes back online, it automatically removes expired roles and catches up on warnings.

Q: Can I modify a timer after it's created?

A: Absolutely! Use /addtime to extend, /removetime to reduce, or /cleartime to delete completely. You can modify at any time.

Q: What if I need to pause a timer temporarily?

A: Use /pausetime to freeze the countdown. The user keeps the role but time doesn't count down. Resume anytime with /resumetime.

Q: How long can I set a timer for?

A: Maximum 10,080 minutes (7 days) per command. But you can use /addtime repeatedly to extend indefinitely.

Q: Can users see their remaining time?

A: Yes! Any server member can use /showtime to check their own or others' timers (with permission).

Q: Does BoostMon have a premium tier?

A: No! All features are completely free. No paid tiers, no ads, no limitations. It's open source on GitHub.

Q: How do warnings work?

A: When you set a timer with a warning channel, BoostMon sends notifications at 60, 10, and 1 minute marks. If no channel specified, warnings are sent as DMs.

Q: What happens if I remove a user's role manually?

A: The timer continues running. When it expires, BoostMon tries to remove the role (no-op if already gone) and marks the timer as expired in the database.

Q: Can I use BoostMon with multiple roles per user?

A: Yes! Each user can have multiple timed roles running simultaneously. Just specify the role when running commands to target specific timers.

Q: Is there an API or webhook integration?

A: Not currently, but BoostMon is designed with extensibility in mind. Contact the developer via GitHub Issues for custom integration requests.

‚ùå Problem: Bot not responding to commands

  • ‚úì Verify BoostMon is online (check member list)
  • ‚úì Ensure it has "Manage Roles" permission in the server
  • ‚úì Check that you're using "/" to trigger slash commands
  • ‚úì Verify the channel isn't restricted for bot usage

‚ùå Problem: Role not being removed after timer expires

  • ‚úì Check if bot has permission to manage the role
  • ‚úì Verify the role isn't protected or positioned higher than bot's role in hierarchy
  • ‚úì Check bot logs for permission errors
  • ‚úì Ensure the role still exists in the server

‚ùå Problem: Warning notifications not sending

  • ‚úì Verify warning channel exists and bot can send messages there
  • ‚úì Check channel permissions for BoostMon
  • ‚úì Try using DMs instead by omitting the #channel parameter
  • ‚úì Verify the channel hasn't been deleted

‚ùå Problem: Timer shows wrong remaining time

  • ‚úì Use /showtime to get accurate timestamp
  • ‚úì Check server timezone matches user timezone
  • ‚úì Verify no system clock skew on the server

‚ùå Problem: Command syntax error

  • ‚úì Verify all required parameters are specified (@user, minutes, @role)
  • ‚úì Check minute values are between 1-10,080
  • ‚úì Ensure you're using the correct command name
